       - Interface in `src/Valt.Infra/Modules/SpendingEvolution/Queries/`
       - Single method:

    2. **SpendingEvolutionQueries.cs**:
       - Implement `ISpendingEvolutionQueries`
       - Inject `ILocalDatabase` in constructor
       - Implement `GetSpendingEvolutionAsync` with the following logic:
         
         a. **Load accounts and categories** (same pattern as TransactionQueries):
            - `var accounts = _localDatabase.GetAccounts().FindAll().ToList();`
            - Filter by visibility: `var visibleAccounts = accounts.Where(a => a.Visible).ToList();`
            - Create lookup dictionaries: `accountDict` by `Id`
            - Get primary currency from settings or first visible account's currency
         
         b. **Build transaction query**:
            - `var query = _localDatabase.GetTransactions().Query();`
            - Filter by date range: `query = query.Where(x => x.Date >= fromDate && x.Date <= toDate)`
            - Filter by category IDs if provided: `query = query.Where(x => categoryIds.Contains(x.CategoryId))`
            - Filter by visible accounts: only include transactions where `FromAccountId` is in visible accounts
            - Only include debit transactions (where `FromFiatAmount < 0` or `FromSatAmount < 0`)
            
         c. **Aggregate by month**:
            - Group transactions by year-month: `new { x.Date.Year, x.Date.Month }`
            - For each group:
              - Fiat total: Sum of absolute `FromFiatAmount` converted to primary currency
              - Sats total: Sum of absolute `FromSatAmount` (only if not null)
              - Transaction count: count of transactions in group
            - IMPORTANT: If a transaction lacks `FromSatAmount`, it still contributes to fiat total but NOT to sats total
            - Track `HasMissingPriceInSats`: true if ANY transaction in the result set has `FromSatAmount == null`
            
         d. **Sort and return**:
            - Order by year, then month ascending
            - Map to `SpendingEvolutionMonthDto` list
            - Return `SpendingEvolutionDataDto` with:
              - `Months = dtoList`
              - `HasMissingPriceInSats = hasMissing`
              - `PrimaryCurrency = primaryCurrency`

       - **Performance optimization** (per D-02 / DATA-07):
         - Use LiteDB query for filtering (date, categories, accounts) — do NOT load all transactions into memory
         - Use `.ToList()` only after filtering
         - Group in memory (LiteDB doesn't support complex aggregation)
         - Target: <500ms for 5 years of data
