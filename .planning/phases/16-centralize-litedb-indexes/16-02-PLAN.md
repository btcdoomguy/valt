# Plan 16-02 — Verify Centralized LiteDB Index Coverage

## Objective

Add automated tests that prove every existing LiteDB index is created immediately after opening an in-memory database, guarding against accidental index loss during the centralization refactor.

Purpose: Prevent silent query-performance regressions by asserting index coverage for every indexed collection.
Output: Two new test fixtures under `tests/Valt.Tests/Infra/DataAccess/`.

## must_haves

- `LocalDatabaseIndexTests` opens an in-memory `LocalDatabase` and asserts `GetIndexes()` contains the expected expressions for all nine indexed collections.
- `PriceDatabaseIndexTests` opens an in-memory `PriceDatabase` and asserts `GetIndexes()` contains the expected expressions for `datasource_bitcoin` and `datasource_fiat`.
- Both test classes clean up their `MemoryStream` instances in `OneTimeTearDown`.
