# Plan 16-01 — Centralize LiteDB Indexes in Database Classes

## Objective

Move every LiteDB `EnsureIndex` call out of per-access collection accessors and into a single private `EnsureIndexes()` method on each database class, invoked from every database open path.

Purpose: Eliminate redundant index-ensuring overhead on hot collection-access paths while preserving all existing indexes.
Output: Refactored `LocalDatabase` and `PriceDatabase` with centralized index creation.

## must_haves

- `LocalDatabase` has a private `EnsureIndexes()` method called from `OpenDatabase`, `OpenInMemoryDatabase`, and `ChangeDatabasePassword`.
- `PriceDatabase` has a private `EnsureIndexes()` method called from `OpenDatabase` and `OpenInMemoryDatabase`.
- Every collection accessor that previously called `EnsureIndex` now returns `GetCollection<T>(name)` directly.
- All index expressions from the 16-RESEARCH.md inventory are preserved in the new `EnsureIndexes()` methods.
