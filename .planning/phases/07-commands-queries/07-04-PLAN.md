<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client -> AssetTools | External AI client invokes tools; all input must be validated by the underlying commands. |
| AssetTools -> Dispatcher | Tools delegate to existing CQRS infrastructure; no direct DB access. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-11 | Tampering | AssetTools.AddLoanStateUpdate / DeleteLoanStateUpdate | mitigate | Delegate to AddLoanStateUpdateCommand/DeleteLoanStateUpdateCommand; reuse existing validators and handlers; do not bypass business rules. |
| T-07-12 | Information Disclosure | AssetTools.GetLoanStateTimeline / GetLatestLoanState | mitigate | Delegate to GetLoanStateTimelineQuery/GetLatestLoanStateQuery; repository lookup scopes results to the supplied AssetId. |
| T-07-13 | Tampering | Missing change notification | mitigate | Mutating tools publish McpDataChangedNotification after successful commands so connected clients see updates. |
| T-07-14 | Spoofing | MCP tool authorization | mitigate | Tools follow existing AssetTools patterns; the MCP server is local and runs only after the database is open. No additional authorization layer is required for this local desktop tool. |
</threat_model>
