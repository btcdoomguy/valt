# Quick Task 20260529-001: Fix copy transaction modal slow opening

## Problem
When using the Copy feature in the Transactions module, the modal takes several seconds to open. This is caused by fetching ALL transactions via `GetTransactionsQuery()` just to find the maximum transaction date.

## Root Cause
In `TransactionEditorViewModel.cs` (lines 510-513), the copy logic does:
```csharp
var allTransactions = await _queryDispatcher!.DispatchAsync(new GetTransactionsQuery());
var lastTransactionDate = allTransactions.Items.Count > 0
    ? allTransactions.Items.MaxBy(t => t.Date)?.Date
    : null;
```

`GetTransactionsAsync` in `TransactionQueries.cs` loads all accounts, categories, filters, maps to DTOs, and loads fixed expense records - extremely expensive for large transaction histories.

## Solution
Add a lightweight `GetLastTransactionDateQuery` that queries LiteDB directly for the maximum date without loading all transactions.

## Tasks

### Task 1: Add GetLastTransactionDate query and handler
- Add `GetLastTransactionDateQuery` in App layer (returns `DateOnly?`)
- Add handler that uses `_localDatabase.GetTransactions().Query().OrderByDescending(x => x.Date).FirstOrDefault()`
- Register in DI

### Task 2: Update TransactionEditorViewModel to use new query
- Replace `GetTransactionsQuery()` call with `GetLastTransactionDateQuery()`
- Keep same date logic (use last date if < today, else today)
